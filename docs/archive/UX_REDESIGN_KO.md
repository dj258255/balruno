# Balruno UX 재설계 가이드 — "3초 원칙"

> 작성일: 2026-04-18
> 중심 원칙: **사용자가 처음 들어와서 3초 안에 "뭘 할 수 있는지" 알아야 한다**
> 대상 사용자: 게임 기획자 (엑셀 익숙, 설명서 싫어함, 빠른 피드백 원함)

---

## 1. 현재 UX의 근본 문제

### 1.1 발견 가능성 (Discoverability) 붕괴
- 16개 플로팅 패널이 어디에 있는지, 뭐가 뭔지 모름
- "Calculator"와 "FormulaHelper"의 차이 설명 없음
- 설정/도움말/가이드가 어디 있는지 탐색 필요

### 1.2 빈 상태 (Empty State) 방치
- 처음 들어오면 빈 화면에 "프로젝트 없음" 만 표시
- "뭘 해야 할지" 가이드 없음
- 신규 유저의 **첫 30초 실패율** 매우 높음

### 1.3 인지 과부하
- 동시에 보이는 선택지가 너무 많음
- 메인 영역 + 사이드바 + 하단 툴바 + 플로팅 패널 = 혼란

### 1.4 도메인 언어 부재
- "시트", "폴더" 같은 일반적 용어 사용
- 게임 기획자가 실제로 쓰는 "Character", "Weapon", "Stage" 부재

### 1.5 상태 인식 어려움
- 지금 어떤 프로젝트를 편집 중인지 명확하지 않음
- 변경사항이 저장됐는지, 다른 사람이 편집 중인지 불명확
- B2B 협업 시 치명적

## 2. 재설계 원칙 — "3초 원칙"

처음 들어온 사용자가 **3초 안에** 다음 3가지를 알 수 있어야 함:

1. **내가 지금 어디 있는가** (맥락)
2. **다음에 무엇을 할 수 있는가** (행동)
3. **도움이 필요하면 어디로 가는가** (지원)

이걸 못하면 재설계 실패.

### 2.1 3초 원칙을 달성하는 4가지 서브 원칙

#### (a) 3개 이상의 Primary Action 금지
화면 당 파란색(primary) 버튼 최대 3개. 나머지는 보조.

#### (b) 레이어 깊이 2단계 초과 금지
클릭 2번 안에 모든 기능 접근 가능. ⌘K가 이걸 완성.

#### (c) 도메인 용어 우선
일반 용어(Sheet, Folder) → 도메인 용어(Characters, Weapons)로 교체.

#### (d) Progressive Disclosure (점진적 공개)
초급 유저에겐 기본 기능만, 숙련 시 고급 드러나게.

### 3.2 상단 바 (60px 고정)

```
┌───────────────────────────────────────────────────────────────┐
│ 🎮 Balruno    🔍 Search... (⌘K)      [👤▼] [?] [🌙]      │
└───────────────────────────────────────────────────────────────┘
```

- **로고**: 클릭 시 홈/대시보드
- **검색바**: ⌘K로도 열림. Linear/Notion 표준
  - 시트/수식/도구/유저/최근 변경 전부 검색
  - 첫 진입 시 placeholder: "무엇이든 검색... 예: DPS, 몬스터, 수식 도움"
- **프로필 드롭다운**: 계정, 팀, 설정
- **?**: 도움말 센터 (Contextual help)
- **🌙**: 다크모드 토글

### 3.3 좌측 사이드바 (224px, 접을 수 있음)

```
┌────────────────┐

│  └ Project B   │

│    🏰 Stages   │
│    💰 Economy  │

└────────────────┘
```

**핵심 변경**:
- 프로젝트 내부 시트를 **도메인 카테고리 아이콘**으로 표시
- 자동 카테고리 분류: 시트 이름에서 감지 (예: "몬스터" → 🎭)
- 또는 유저가 카테고리 직접 할당 가능

### 3.4 View 탭 (48px)

```
┌──────────────────────────────────────────────────────────┐
│ [프로젝트명 ▼]  |  [Grid] [Board] [Dashboard]  |  [공유] │
└──────────────────────────────────────────────────────────┘
```

**Grid View** (기본):
- Excel-like 스프레드시트
- 기존 SheetTable 컴포넌트 재사용

**Board View** (monday style):
- 상태 컬럼 기준 칸반
- "Draft / 검토중 / 승인됨 / 라이브" 자동 그룹

**Dashboard View** (디렉터용):
- KPI 위젯 (파워 커브, 승률 분포, 밸런스 변경 추이)
- 읽기 전용 기본

### 3.5 하단 도구 레일 (40px)

```
┌──────────────────────────────────────────────────────────┐
│ 🔧 [수식] [시뮬] [분석] [Export]      📊 Draft ✓ 저장됨   │
└──────────────────────────────────────────────────────────┘
```

**핵심 변경**:
- 16개 패널 → 4개 카테고리
- 카테고리 클릭 시 하단에 부속 패널 펼침
- 상태 뱃지 (Draft / 저장 상태)

**카테고리 매핑**:
- **수식**: FormulaHelper + Calculator + FormulaPresetPicker
- **시뮬**: SimulationPanel + DPSVariancePanel + EconomyPanel
- **분석**: ImbalanceDetectorPanel + BalanceAnalysisPanel + GoalSolverPanel + CurveFittingPanel + ComparisonChart + GrowthCurveChart + BalanceValidator
- **Export**: ExportModal + ImportModal + EntityDefinition + TemplateSelector

## 4. 핵심 신규 UX 패턴

## 5. 디자인 언어 (Visual Design)

### 5.1 방향: **"조용한 전문성"**

참고 제품:
- **Linear** — 미니멀, 키보드 퍼스트
- **Raycast** — ⌘K 중심, 쉬운 팔레트
- **Arc Browser** — 여백 활용, 부드러운 인터랙션

이 세 개의 공통점:
- 색상 최소 (그레이 85% + primary 1개)
- 타이포그래피 중심
- 여백이 고급감 전달
- 애니메이션 짧고 목적 있음

### 5.2 색상 팔레트

```
Primary:     #6366F1 (indigo-500) — 메인 액션, 브랜드
Success:     #10B981 (emerald-500) — 저장됨, 승인됨
Warning:     #F59E0B (amber-500) — 검토 필요
Danger:      #EF4444 (red-500) — 롤백, 삭제
Info:        #3B82F6 (blue-500) — 알림

그레이 스케일 (사용률 85%):
  50/100/200/300/400/500/600/700/800/900
```

### 5.3 타이포그래피

```
한국어: Pretendard Variable
영어: Inter Variable

크기 스케일:
  text-xs (12px)  — 메타 정보, 타임스탬프
  text-sm (14px)  — 보조 텍스트
  text-base (15px) — 본문 (현재 16px → 살짝 줄임)
  text-lg (18px)  — 섹션 헤더
  text-xl (20px)  — 페이지 타이틀
  text-2xl (24px) — 주요 대시보드 숫자
```

### 5.4 아이콘

- **lucide-react 단일 유지** (이미 사용 중)
- 굵기 `stroke-width="1.5"` 통일
- 크기 16/20/24 세 단계만 사용

### 5.5 스페이싱

```
현재: 4, 8, 12, 16, 24, 32
제안: 4, 8, 12, 16, 24, 32, 48, 64

여백을 더 넓게 → 고급감 증가
압축된 UI는 "저렴해 보임"
```

### 5.6 그림자 & 모서리

```
Shadow:
  none (기본)
  sm (0 1px 2px rgba) — 살짝 뜨는 요소
  md (0 4px 8px rgba) — 모달, 팝오버
  lg — 드래그 중인 요소만

Border-radius:
  4px — 입력 필드, 작은 버튼
  6px — 카드, 패널 (8px → 6px로 축소, 덜 카툰)
  12px — 모달
  9999px — 원형 (아바타, 뱃지)
```

### 5.7 애니메이션

원칙: **목적이 있는 움직임만**

```
✅ 허용:
  - 모달 등장 (fade + scale 0.95→1, 150ms)
  - 툴팁 (fade, 100ms)
  - 페이지 전환 (Next.js 기본)
  - 저장됨 체크 마크 (spring, 300ms)

❌ 금지:
  - 스크롤 페러랙스
  - 호버 시 과한 움직임
  - 로고 로테이션 등 장식성
```

## 7. 접근성 (Accessibility)

B2B는 접근성 법규 요구 많음:

### 7.1 필수 체크리스트
- [ ] 모든 버튼에 `aria-label` 또는 텍스트
- [ ] 키보드만으로 모든 기능 사용 가능
- [ ] 대비율 WCAG AA 이상 (텍스트 4.5:1)
- [ ] 포커스 인디케이터 명확 (2px outline)
- [ ] 스크린리더 친화 (`role`, `aria-live` 등)

### 7.2 국제화 (i18n)
- 현재 EN/KO → **일본어/중국어 간체 추가 권고**
- 날짜 포맷 지역별 자동
- RTL 미지원 (아랍어 불필요)
