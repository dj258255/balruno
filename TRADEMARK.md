## What you CAN do

- Use the names "PowerBalance" / "balruno" in **factual references** to
  the upstream project (e.g. "based on PowerBalance", "fork of balruno").
- Reuse the **codebase** under its respective license.
- Build your own product using the code, with **your own name and brand**.
- Reference the project in academic / journalistic contexts.

## What you CANNOT do (without written permission)

- Name your fork, hosted service, or derivative product **"PowerBalance"**,
  **"balruno"**, or any confusingly similar name (e.g. "PowerBalance Cloud",
  "balruno Pro", "PowerBalance for X").
- Use the **project logo** in your own product's branding, marketing, or UI.
- Register **domain names** containing "powerbalance" or "balruno".
- Imply official endorsement, partnership, or affiliation when there is none.

## Modification

If you fork the project, you must **change the name and logo** in any
public-facing context (website title, app name, marketing materials,
package name on registries).

You may keep references in source code comments, license files, and
attribution notices.

## Why Trademark Protection?

Trademark protection ensures users can trust that "PowerBalance" /
"balruno" refers to the official project. Without it, a malicious fork
could distribute compromised code under the same name, eroding user trust.
This is standard practice for OSS projects (Mozilla, Linux Foundation,
Apache, etc).
